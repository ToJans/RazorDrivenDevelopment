RDD - Razor driven development - The first bankaccount specs

This is an example of a BDD/CQRS approach using the following tools:

  • An empty ASP.Net project
  • FakeItEasy
  • ShouldLy
  • Razor templates

Works pretty sweet IMO...

    **************
    * DEPOSITING *
    **************

     Deposit some money
     ------------------

        Given a bankaccount                                     
        When we deposit an amount                               
        Then the amount should have been deposited              

     Deposit a negative amount
     -------------------------

        Given a bankaccount                                     
        When we deposit a negative amount                       
        Then the amount should have been deposited              

    ***************
    * WITHDRAWING *
    ***************

     Withdraw some money
     --------------------

        Given a bankaccount with some money on it               
        When we withdraw a small amount from the account        
        Then the small amount should have been credited         

     Withdrawing a negative amount should fail
     ------------------------------------------

        Given a bankaccount with some money on it               
        When we withdraw a negative amount from the account     
        Then the negative amount should not have been withdrawn 

     Withdrawing an amount that is to large should fail
     --------------------------------------------------

        Given a bankaccount with some money on it               
        When we withdraw an amount that is to large             
        Then the to large amount should not have been withdrawn
